Posted on 2019-04-04 | In 前端

Javascript模块化

优点 缺点 代码示例
原始文件 None 1. 污染全局作用域
2. 依赖关系不明显，难以维护		 <script src="jquery.js"></script>
<script src="jquery_scroller.js"></script>
<script src="main.js"></script>
<script src="other1.js"></script>
<script src="other2.js"></script>
<script src="other3.js"></script>
CommonJS规范 实现了服务端的模块化加载 1. 同步加载，只使用于服务端 var x = 5;
var addX = function (value) {
return value + x; };
module.exports.x = x;
module.exports.addX = addX;
AMD规范 实现了浏览器端的模块化加载 1. 提高了开发成本
2. 无法按需加载，必须提前加载好所有依赖		 define(['./a', './b'], function(a, b) {
a.doSomething()
b.doSomething() });
CMD规范 1.实现了浏览器端的模块化加载
2.可以按需加载，依赖就近		 依赖SPM打包，模块的加载逻辑偏重 define(function(require, exports, module) {
var a = require('./a')
a.doSomething()
var b = require('./b') // 依赖可以就近书写
b.doSomething() });
ES6模块化 1. 非社区规范，真正的JS规范
2.简单强大		 需要Babel支持 import store from '../store/index'
import {mapState, mapMutations, mapActions} from 'vuex'

import util from '../utils/js/util.js'
